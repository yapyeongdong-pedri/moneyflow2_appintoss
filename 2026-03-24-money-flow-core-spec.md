# Money Flow 핵심 기능명세서 (Core)

- 문서 버전: v1.0-core
- 작성일: 2026-03-24
- 목적: 기획/개발/QA가 빠르게 합의할 수 있는 최소 핵심 기준

## 1. 제품 한 줄 정의
월급통장을 시작점으로 계좌/카드/지출항목의 자금 흐름을 노드 그래프로 관리하는 모바일 중심 앱.

## 1.1 모바일 최적화 원칙 (필수)
1. 본 제품은 모바일 웹앱이므로 모바일 최적화는 선택이 아니라 필수 요구사항이다.
2. 모든 화면은 모바일 뷰포트 기준에서 가독성과 조작성을 우선해야 한다.
3. 타이포/간격/버튼/입력요소 크기는 모바일에서 오동작 없이 읽고 누를 수 있도록 조정되어야 한다.
4. 작은 높이 화면에서도 텍스트 잘림/버튼 줄바꿈/중요 정보 가려짐이 없어야 한다.
5. 기능 구현 시 로직 완성도와 동일한 수준으로 UI 스케일 적합성(글자 및 요소 사이즈)이 중요하다.
6. 핵심 편집/조회 화면은 스크롤 없이 한 화면에서 동작해야 한다(무스크롤 원칙).

## 2. 핵심 기능(반드시 포함)
1. 인트로 -> 메인 진입
2. 노드 추가/수정/삭제
- 계좌, 카드, 지출항목 추가
- 월급통장은 삭제 불가
3. 자동 연결/검증
- 허용된 연결만 생성
4. 자동 레이아웃
- 월급통장 고정
- 계좌 정렬 순서 및 지출 2줄 규칙 적용
5. 저장/복원
- localStorage 자동 저장/재진입 복원
6. 공유
- Toss -> Web Share -> Clipboard 순 폴백

## 2.1 MVP 범위 고정
1. 현재 MVP는 \"흐름 구조 시각화\"에 집중한다.
2. 구체적인 금액 입력/합계/정산/분석 기능은 이번 버전에 포함하지 않는다.
3. 금액 관련 기능은 후속 버전(Phase 2+)에서 별도 스펙으로 확장한다.

## 3. 노드/흐름 규칙
### 3.1 노드 타입
- 월급통장(`salary_account`)
- 계좌(`asset_account`)
- 카드(`payment_instrument`)
- 지출항목(`expense_category`)

### 3.2 계좌 구분(표시 순서 고정)
- 입출금 -> 적금(지출용) -> 투자 -> 연금 -> 적금(저축용)

### 3.3 허용 연결
- 월급통장 -> 계좌/카드/지출항목
- 계좌 -> 계좌/카드/지출항목
- 카드 -> 지출항목

### 3.4 연결 인터랙션 원칙
1. 연결 생성은 클릭-클릭 방식으로 한다.
2. A 노드 클릭 후 B 노드를 클릭하면 `A -> B` 연결 생성 시도를 수행한다.
3. 연결 생성 시 허용 연결 규칙(3.3)을 즉시 검증한다.
4. 허용되지 않는 조합이면 연결은 생성하지 않고 안내 메시지를 노출한다.
5. 연결 해제는 선(엣지) 클릭으로 선택 후 해제 액션으로 수행한다.
6. 선 선택 상태에서는 대상 선을 시각적으로 명확히 강조해야 한다.

## 4. 배치 핵심 규칙
1. 계좌는 subtype 순서를 유지한 채 가운데 정렬
2. 지출항목은
- 1~7개: 1줄 중앙
- 8~14개: 2줄(7+나머지)
3. 초기화 후에도 월급통장 위치는 레이아웃 기준 좌표 유지

## 5. 상한/입력 제약
- 계좌 행별 최대 7
- 카드 최대 5
- 지출항목 최대 14
- 노드명 최대 30자
- 메모 최대 40자

## 6. UI 핵심 스타일 기준
- 노드: 화이트 베이스 + 좌측 accent bar
- 노드 내부 텍스트: `#3f6175`
- 흐름선/화살표/엣지 텍스트: `#3f6175`
- 선(엣지) 화살표는 단순한 단일 스타일로 유지해 복잡도를 낮춘다.
- 과한 장식(다중 화살표 스타일, 과도한 애니메이션)은 사용하지 않는다.

## 6.1 밀도 제어(요소 과다 방지)
1. 항목이 많아지는 상황에서도 한 화면에 요소가 과도하게 늘어나지 않도록 UI 밀도를 제어한다.
2. 다건 입력/선택은 드롭다운(드롭박스), 세그먼트, 컴팩트 버튼 우선으로 설계한다.
3. \"항상 펼쳐진 긴 폼\"보다 \"단계적 입력\"과 \"요약형 컨트롤\"을 우선 적용한다.
4. 버튼은 의미 구분은 유지하되 면적을 과도하게 차지하지 않도록 컴팩트 사이즈를 기본값으로 한다.

## 6.2 파괴적 액션 확인 팝업(필수)
아래 액션은 실행 전에 반드시 확인 팝업을 띄운다.
1. 노드 삭제
2. 선(엣지) 삭제/연결 해제
3. 전체 초기화

확인 팝업 공통 규칙:
- 버튼은 `확인/취소` 2개
- 기본 포커스는 `취소`
- 확인 전에는 실제 데이터 변경 금지
- 메시지는 결과를 명확히 고지
  - 예: \"이 노드를 삭제하면 연결된 선도 함께 삭제됩니다. 진행할까요?\"
  - 예: \"선 연결을 해제할까요?\"
  - 예: \"데이터가 모두 삭제됩니다. 초기화할까요?\"

## 7. 수용 기준(AC)
- AC-01: 인트로에서 메인으로 정상 전환
- AC-02: 허용되지 않은 연결 생성 차단
- AC-03: 계좌 순서가 항상 지정 순서 유지
- AC-04: 지출항목 8개부터 2줄 배치
- AC-05: 카드 6개째, 지출 15개째 추가 차단
- AC-06: 새로고침 후 상태 복원
- AC-07: 공유 실패 시 폴백 경로 동작

## 8. 비범위(이번 버전 미포함)
- 서버 동기화
- 멀티 유저 협업
- 통계/리포트 분석 화면
- Undo/Redo UI 버튼
