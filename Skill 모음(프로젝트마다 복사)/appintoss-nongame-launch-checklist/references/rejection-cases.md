# 실제 반려 사례 상세 체크리스트

실제 앱인토스 심사에서 반려된 사례들을 기반으로 추출한 점검 항목이다.
각 카테고리별로 반려 빈도가 높은 순서로 정리되어 있다.

---

## 네비게이션 반려 사례

| 반려 사유 | 설명 |
|-----------|------|
| 자체 백버튼 사용 | 공통 내비게이션 백버튼 대신 자체 뒤로가기 버튼 구현 |
| 히스토리 백 미동작 | 공통 내비게이션 백버튼 터치 시 이전 페이지로 돌아가지 않음 |
| 미니앱 미종료 | 최초 화면에서 백버튼 터치 시 리프레시만 되고 종료되지 않음 |
| 앱 로고 미노출 | 공통 내비게이션에 앱 아이콘/로고가 표시되지 않음 |
| 자체 헤더 사용 | 앱 상단에 자체 구현한 헤더가 있어 인앱 브라우저처럼 보임 |
| 햄버거 메뉴 사용 | 자체 햄버거 메뉴를 공통 내비게이션 커스텀 대신 사용 |
| 커스텀 버튼 오동작 | 즐겨찾기(하트) 등 커스텀 버튼이 제대로 동작하지 않음 |
| 기능 스킴 내비게이션 오류 | 기능 스킴으로 접속 시 내비게이션 설정이 정상 반영되지 않음 |

**핵심 원칙:** 내비게이션 영역은 앱인토스 브랜드 인지 핵심 영역이므로, 자체 구현을 모두 제거하고 공통 컴포넌트만 사용해야 한다.

---

## 로그인/인증/권한 반려 사례

| 반려 사유 | 설명 |
|-----------|------|
| 시작 즉시 로그인 | 인트로 뷰 없이 앱 시작 직후 로그인 화면 노출 |
| 자체 이용약관 | 토스 로그인 플로우 약관 대신 자체 약관 동의 화면 구현 |
| 연동 해제 미처리 | 토스 로그인 연동 해제 콜백을 받아 로그아웃 처리하지 않음 |
| 재로그인 실패 | 연동 해제 후 재로그인 플로우가 정상 동작하지 않음 |
| 권한 바텀싯 미구현 | 위치/카메라 등 권한 요청 시 토스 가이드 기반 바텀싯 미사용 |
| 미사용 권한 표시 | 권한 목록에 실제로 요청하지 않는 권한이 포함됨 |
| 탈퇴 기능 오류 | 탈퇴하기 기능 구현했으나 정상 동작하지 않음 |

**핵심 원칙:**
- 로그인은 반드시 인트로 뷰(서비스 소개 화면) 이후에 진행
- 자체 이용약관 제거, 토스 로그인 플로우에서 약관 동의
- 콜백 URL 등록하여 연동 해제 시 로그아웃 처리

## 스킴/라우팅 반려 사례

| 반려 사유 | 설명 |
|-----------|------|
| 기능 스킴 404 | 콘솔에 등록한 기능 스킴 URL이 접속 시 404 오류 |
| 랜딩 페이지 오류 | 기능 스킴 랜딩 페이지가 정상 렌더링되지 않음 |

## UI/UX 반려 사례

| 반려 사유 | 설명 |
|-----------|------|
| 바텀싯 닫기 불가 | 바텀싯이 전체 화면을 덮어 닫기 버튼 터치 불가능 |
| 불필요한 좌우 스크롤 | 홈 화면에서 의도하지 않은 좌우 스크롤 발생 |
| 데이터 미반영 | 좋아요/즐겨찾기 후 데이터가 즉시 반영되지 않음 |
| 소진 안내 부재 | 횟수 제한 기능(뽑기 등) 소진 후 안내 메시지 없음 |
| 기능 버튼 오류 | 이미지 저장 등 주요 기능 버튼이 동작하지 않음 |
| 과도한 애니메이션 | 배너 등 화면 요소에 과도한 깜빡임/애니메이션 |
| 핀치줌 활성화 | 확대/축소 핀치줌이 비활성화되지 않음 |
| TDS 모달 미사용 | 앱 내 얼럿에 기본 alert/confirm 대신 TDS 모달 미적용 |
| 탭바 형태 오류 | 탭바가 플로팅 형태가 아닌 고정 형태로 구현됨 |
| 탭바 크기 부적절 | 탭바 여백, 아이콘/문구 크기가 가이드와 맞지 않음 |
| 광고 로딩 오류 | 인앱 광고 로딩이 정상 동작하지 않음 |
| 재화 소모 오류 | 재화 소모 기능(코인 차감 등)이 정상 동작하지 않음 |

## 텍스트/브랜딩 반려 사례

| 반려 사유 | 설명 |
|-----------|------|
| 앱 이름 불일치 | 앱 이름이 콘솔 정보등록과 다르게 노출됨 |
| 브랜드명 불통일 | 공유하기, 메타 태그 등에서 서로 다른 브랜드명 사용 |
| 구독 오인 워딩 | 구독 상품처럼 오해될 수 있는 워딩 포함 |
| 결제 금액 불일치 | UI 표시 금액과 실제 결제 금액이 다름 |

---

## 반려 시 대응 전략

1. **반려 사유 확인**: 콘솔에서 반려 사유를 정확히 확인
2. **해당 항목 수정**: 이 체크리스트 기준으로 관련 항목 수정
3. **연관 항목 점검**: 같은 카테고리의 다른 항목도 함께 점검
4. **재심사 제출**: 수정 완료 후 재심사 요청

> 한 번에 여러 항목이 반려될 수 있으므로, 제출 전 전체 체크리스트를 빠짐없이 점검하는 것이 효율적이다.
