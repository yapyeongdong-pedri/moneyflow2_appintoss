# 앱인토스 비게임 미니앱 출시 전 최종점검

앱인토스 비게임 미니앱의 출시 검수를 통과하기 위한 종합 체크리스트다.
실제 반려 사례와 공식 출시 가이드를 결합하여, 심사 제출 전 누락 항목이 없도록 점검한다.

## 점검 워크플로우

아래 체크리스트를 복사하여 진행 상황을 추적하라:

```
출시 전 최종점검 진행현황:
- [ ] 1단계: 접속 및 앱 내 기능 점검
- [ ] 2단계: 내비게이션 바 점검
- [ ] 3단계: 로그인/인증/권한 점검
- [ ] 4단계: 스킴/라우팅 점검
- [ ] 5단계: UI/UX 점검
- [ ] 6단계: 텍스트/브랜딩 점검
- [ ] 7단계: 결제 점검 (인앱결제/토스페이)
- [ ] 8단계: 인앱 광고 점검
- [ ] 9단계: 외부 링크/앱 설치 유도 정책 점검
- [ ] 10단계: TDS 디자인 시스템 점검
- [ ] 11단계: 공유 리워드 점검
```

---

## 1단계: 접속 및 앱 내 기능 (필수)

- [ ] 메인 스킴 URL 접속 시 미니앱이 정상 로딩되는가?
- [ ] 콘솔에 등록한 모든 '앱 내 기능'이 미니앱 안에서 완결성 있게 동작하는가?
- [ ] 서비스가 외부 앱/브라우저 이동 없이 미니앱 내에서 완료 가능한가?

---

## 2단계: 내비게이션 바 (필수)

> 내비게이션 바는 모든 화면 상단에 반드시 배치되는 앱인토스 공통 UI 컴포넌트다.

- [ ] 자체 구현 백버튼/뒤로가기 버튼을 **모두 제거**했는가?
- [ ] **공통 내비게이션 백버튼**으로 히스토리 백이 동작하는가?
- [ ] 최초 화면(온보딩/랜딩)에서 백버튼 터치 시 **미니앱이 종료**되는가? (리프레시만 되고 종료 안 되면 반려)
- [ ] **공통 내비게이션 앱 아이콘/로고**가 정상 노출되는가?
- [ ] 앱 상단에 **자체 헤더**가 없는가? (인앱 브라우저처럼 보이면 반려)
- [ ] **자체 햄버거 메뉴를 제거**하고 공통 내비게이션 커스텀을 활용하는가?
- [ ] 액세서리 버튼은 **모노톤 아이콘**만 사용하는가? (컬러 아이콘 불가)
- [ ] 액세서리 버튼은 **1개만** 설정했는가?
- [ ] 커스텀 버튼(즐겨찾기 하트 등)이 정상 동작하는가?
- [ ] 기능 스킴 접속 시 공통 내비게이션 및 커스텀 설정 버튼이 정상인가?

**구현 참고:**
```typescript
// Web: defineConfig
navigationBar: {
  withBackButton: true,
  withHomeButton: true,
  initialAccessoryButton: {
    id: 'heart',
    title: 'Heart',
    icon: { name: 'icon-heart-mono' },
  },
}
```

## 3단계: 로그인/인증/권한 (기능 사용 시 필수)

- [ ] 토스 로그인은 **인트로 뷰 이후**에 진행되는가? (앱 시작 직후 로그인 화면 노출 금지)
- [ ] **자체 이용약관을 제거**하고 토스 로그인 플로우에서 약관 동의를 받는가?
- [ ] 토스 로그인 연동 해제 시 **콜백을 통해 로그아웃 처리**가 되는가?
- [ ] 연동 해제 후 **재로그인 플로우**가 정상 진행되는가?
- [ ] 위치/카메라 등 권한 필요 시 **토스 권한 문서 기반 바텀싯**을 구현했는가?
- [ ] 권한 목록에 표시된 권한이 **실제로 요청**되는가?
- [ ] 권한 거부 시 **재요청 또는 안내 방법**이 있는가?
- [ ] **탈퇴하기 기능** 구현 시 정상 동작하는가?

**토스 로그인 핵심 규칙:**
- `appLogin()` → 인가 코드(유효 10분, 일회성) → 서버에서 토큰 교환
- Toss OAuth AccessToken 유효: 1시간 / RefreshToken 유효: 14일
- **Toss OAuth 토큰**(AccessToken, RefreshToken)은 반드시 서버에서만 사용하고 클라이언트에 노출하지 않는다
- 서버가 Toss OAuth 토큰으로 사용자 정보를 조회한 뒤, **앱 자체 JWT를 발급**하여 클라이언트에 반환하는 패턴이 표준이다
- 앱 자체 JWT를 클라이언트(localStorage 등)에 저장하는 것은 일반적인 관행이며, 출시된 앱들도 동일한 패턴을 사용한다
- 콜백 URL 콘솔 등록 필수 (referrer: UNLINK, WITHDRAWAL_TERMS, WITHDRAWAL_TOSS)

## 4단계: 스킴/라우팅 (필수)

- [ ] 등록한 모든 **기능 스킴 URL이 404 없이 정상 동작**하는가?
- [ ] 메인 스킴 외 **기능 스킴 랜딩 페이지**가 정상 렌더링되는가?

---

## 5단계: UI/UX (필수)

- [ ] 바텀싯이 화면 전체를 덮어 **닫기 버튼이 터치 불가능**한 경우가 없는가?
- [ ] 홈 화면에서 **불필요한 좌우 스크롤**이 비활성화되어 있는가?
- [ ] 좋아요/즐겨찾기 등 **기능 동작 후 데이터가 정상 반영**되는가?
- [ ] 횟수 제한 기능(랜덤 뽑기 등)이 **소진 후에도 정상 안내**되는가?
- [ ] 이미지 저장 등 **주요 기능 버튼이 정상 동작**하는가?
- [ ] 화면 요소(배너 등)에 **과도한 깜빡임/애니메이션**이 없는가?
- [ ] **확대/축소 핀치줌**이 비활성화되어 있는가?
- [ ] 앱 내 **얼럿이 네이티브 `alert()`/`confirm()` 대신 커스텀 모달** (TDS 모달 또는 자체 구현)로 적용되어 있는가?
- [ ] **재화 소모 기능**이 정상 동작하는가?

---

## 6단계: 텍스트/브랜딩 (필수)

- [ ] **앱 이름**이 `granite.config.ts`(또는 `bedrock.config.ts`)의 `brand.displayName`과 정확히 일치하는가?
- [ ] `index.html`의 `<title>`, `<meta og:title>` 등에서 앱 이름이 **displayName과 동일**(공백 포함)한가?
- [ ] 앱 이름이 **콘솔 정보등록과 동일**하게 노출되는가?
- [ ] 공유하기, 메타 태그 등에서 **브랜드명이 통일**되어 있는가?
- [ ] **구독 상품** 워딩이 포함되어 있지 않은가? (구독 오인 워딩 제거)
- [ ] UI에 표시된 **결제 금액**과 실제 결제 금액이 일치하는가?

**브랜딩 규칙:**
- 로고: 600x600px 각진 정사각형, 라이트/다크 모드 모두 가시적 배경 필수
- 이름: 한글 작성 권장 (`토스` O, `Toss` X)
- 컬러: `brand.primaryColor`에 `#` 포함 6자리 헥스코드 (예: `#3182F6`)

---

## 7단계: 결제 점검 (기능 사용 시 필수)

### 인앱 결제 (IAP)
- [ ] 소모성/비소모성 아이템 구분이 올바른가?
- [ ] 결제 진행 중 앱 내 기능(음악, 영상 등)이 일시 정지되는가?
- [ ] 결제 완료 후 자동으로 기능이 재개되는가?
- [ ] 상품 이름, 수량, 총 결제 금액이 명확히 안내되는가?

### 토스페이
- [ ] 앱인토스 전용 토스페이 가맹점 키를 발급받았는가? (기존 키 사용 불가)
- [ ] 주문 번호가 중복 사용되지 않는가?
- [ ] 결제 전 상품 이름, 수량, 총 금액, 할인, 환불 규정을 안내하는가?

---

## 8단계: 인앱 광고 점검 (기능 사용 시 필수)

> IntegratedAd v2 API가 현재 표준이다. 레거시 AdMob API(`loadAppsInTossAdMob`)도 아직 동작하지만 v2 사용을 권장한다.

### IntegratedAd v2 (권장)
- [ ] 전면 광고: `loadFullScreenAd` → `showFullScreenAd` 순서로 호출하는가?
- [ ] 배너 광고: `TossAds.attachBanner`로 마운트하는가?
- [ ] **광고 로딩**이 정상적으로 동작하는가?
- [ ] 광고 이벤트 핸들링이 올바른가? (loaded, show, impression, clicked, dismissed, failedToShow)
- [ ] 보상형 광고의 경우 `userEarnedReward` 이벤트를 처리하는가?

### Legacy AdMob API (하위 호환)
- [ ] `loadAppsInTossAdMob` → `showAppsInTossAdMob` 순서로 호출하는가?

## 9단계: 외부 링크/앱 설치 유도 정책 (필수)

> 위반 시 콘텐츠 비노출 또는 서비스 이용 제재 가능

### 자사 앱 설치 유도 금지
- [ ] 앱 설치를 직접 유도하는 문구가 없는가?
- [ ] 앱 설치를 유도하는 이미지/배너가 없는가?
- [ ] 앱 마켓 링크가 삽입되어 있지 않은가?
- [ ] 앱 설치 관련 혜택 안내가 없는가?

### 외부 링크 제한
- [ ] 앱 내 기능으로 등록한 서비스를 미니앱 내에서 모두 경험할 수 있는가?
- [ ] 외부 결제창으로 이동하지 않는가?
- [ ] 공유하기 링크가 자사 웹사이트가 아닌 토스 공유 링크로 랜딩되는가?
- [ ] 주요 기능이 외부 링크에 의존하지 않는가?

**허용되는 외부 링크:** 법률 고지, 공공기관/제휴기관 공식 페이지, 타사 정보 확인

상세 가이드: [references/external-link-rules.md](references/external-link-rules.md)

## 10단계: TDS 디자인 시스템 (권장)

> TDS 사용은 **권장 사항**이며, 미설치가 직접적인 반려 사유는 아니다.
> 실제 출시된 비게임 미니앱 중 TDS 패키지 없이 자체 UI로 심사를 통과한 사례가 다수 존재한다.
> 다만, TDS를 사용하면 토스 앱과의 UX 일관성이 높아져 심사에 유리할 수 있다.

- [ ] TDS 컴포넌트를 사용하거나, 자체 UI가 토스 앱과 시각적으로 어울리는가?
- [ ] **탭바**가 필요한 경우 토스 제공 **플로팅 형태** 또는 유사한 패턴으로 구현했는가?
- [ ] 탭 개수가 2~5개 이내인가?
- [ ] 탭바 여백 및 아이콘/문구 크기가 적절한가?

**TDS 패키지 (사용 시 참고):**
| 플랫폼 | 프레임워크 버전 | 패키지 |
|--------|----------------|--------|
| React Native | `@apps-in-toss/framework` < 1.0.0 | `@toss-design-system/react-native` |
| React Native | `@apps-in-toss/framework` >= 1.0.0 | `@toss/tds-react-native` |
| Web | `@apps-in-toss/web-framework` < 1.0.0 | `@toss-design-system/mobile` |
| Web | `@apps-in-toss/web-framework` >= 1.0.0 | `@toss/tds-mobile` |

## 11단계: 공유 리워드 (기능 사용 시 필수)

- [ ] 공유 링크가 `getTossShareLink`로 생성되는가?
- [ ] 공유하기가 `share` API를 통해 동작하는가?
- [ ] 공유 결과 화면이 정상 렌더링되는가?

## 최종 확인

모든 단계를 점검한 후:

1. **필수 항목(1~6, 9단계)**: 하나라도 미통과 시 반려
2. **권장 항목(10단계 TDS)**: TDS 미사용이 직접 반려 사유는 아니나, 토스 UX 일관성을 위해 권장
3. **기능별 항목(7~8, 11단계)**: 해당 기능 사용 시 필수
3. 체크리스트를 기준으로 미통과 항목을 리포트하고 수정 방안 제시

상세 반려 사례: [references/rejection-cases.md](references/rejection-cases.md)
