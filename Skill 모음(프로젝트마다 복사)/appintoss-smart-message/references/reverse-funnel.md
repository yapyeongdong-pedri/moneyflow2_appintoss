# 역퍼널 브레인스토밍

아직 앱 개발 전인 아이디어 단계 사용자를 위한 프로세스.
"앱 만들기 전에 메시지부터 쓰세요" 원칙에 기반한 메시지→기능 역설계.

---

## 프로세스

### Step 1: 핵심 질문

사용자에게 묻는다:
"토스 유저가 이 푸시를 받았을 때, 한 문장으로 뭘 기대하고 클릭할까요?"

### Step 2: 3가지 메시지 콘셉트 제안

사용자의 답변을 기반으로 3가지 방향의 메시지 콘셉트를 제안한다.

예시 (심리 테스트 앱):
- 콘셉트 A — 호기심형: "나의 숨은 성격 유형 / 토스에서 5개 질문에 답해보세요"
- 콘셉트 B — 보상형: "성격 테스트하고 9원 받기 / 토스에서 결과도 확인해요"
- 콘셉트 C — 관계형: "나와 잘 맞는 사람은 누구 / 토스에서 궁합 테스트 해보세요"

### Step 3: 메시지→기능 역설계

사용자가 하나를 고르면, 그 메시지가 클릭되기 위해 앱에 반드시 있어야 할 기능을 역으로 정리한다.

예시 (콘셉트 A 선택 시):
- "5개 질문" → 질문 5개로 구성된 테스트 화면 필요
- "숨은 성격 유형" → 결과 유형 분류 로직 필요
- "토스에서" → 토스 앱 내 미니앱으로 구현

### Step 4: 역설계 결과 확인

메시지→기능 역설계 결과를 사용자에게 보여주고 확인받는다.

```
[메시지→기능 역설계]
선택한 메시지: "나의 숨은 성격 유형 / 토스에서 5개 질문에 답해보세요"

앱에 필요한 기능:
1. 질문 5개 → 테스트 화면 (질문/선택지 UI)
2. 성격 유형 분류 → 결과 로직 (N개 유형)
3. 결과 화면 → 유형 설명 + 공유 기능
4. (선택) 프로모션 → 토스 포인트 지급 연동

이 기능들이 앱에 포함되면 이 메시지로 마케팅할 수 있어요.
맞나요? 수정할 부분이 있으면 알려주세요.
```

확인되면 Phase 2의 정보 확정 단계로 복귀한다.
