# 상세 카피라이팅 가이드

토스 공식 웨비나 기반 A/B 데이터, Winner/Loser 사례, 짧게 쓰기 테크닉 상세.

---

## 타이틀↔본문 연결 공식 — 완성 소재의 핵심

타이틀과 본문은 독립적인 두 문장이 아니라 **하나의 완성된 대화**다. 타이틀이 미끼(hook)를 던지고, 본문이 낚시줄(answer)을 건다.

### 왜 이것이 가장 중요한가

아무리 글자수를 맞추고, 반려 규칙을 지키고, 쉬운 말로 써도, 타이틀과 본문이 **같은 말을 반복**하거나 **핵심 정보가 빠져있으면** 유저는 클릭하지 않는다. "토스에서 뭐 눌러요"는 반려되지 않지만, 아무도 누르지 않는다.

### 3가지 연결 공식

| 공식 | 타이틀 역할 | 본문 역할 | 완성 소재 예시 |
|------|-----------|----------|-------------|
| **보상→행동** | 보상을 보여줌 | "어떻게 받는지" 알려줌 | "매일 9원 받는 습관" / "토스에서 체크만 하면 돼요." |
| **행동→보상** | 행동을 보여줌 | "뭘 받는지" 알려줌 | "습관 3번 체크" / "토스에서 체크하면 9원 받아요." |
| **호기심→행동** | 궁금증 유발 | "뭘 하면 되는지" 알려줌 | "판다가 배고프대요" / "토스에서 판다에게 밥 줘보세요." |

### 앵글별 완성 소재 공식 (타이틀+본문 세트)

건강습관 앱(판다 캐릭터, 습관 체크 3번, 프로모션 9원) 기준:

**호기심 앵글 — "뭔데?" 유발 → 행동 안내:**
- "판다가 배고프대요" / "토스에서 판다에게 밥 줘보세요." (궁금 → 행동)
- "건강카드가 도착했어요" / "토스에서 열어보면 9원 받아요." (궁금 → 보상)

**행동묘사 앵글 — 손가락 동작 → 보상 연결:**
- "버튼 3번 눌러 9원" / "토스에서 습관 체크해 보세요." (행동+보상 → 구체적 안내)
- "체크 3번이면 끝" / "토스에서 누르면 9원 받아요." (쉬운 행동 → 보상)

**확실한 보상 앵글 — 확정 금액 → 행동 안내:**
- "매일 9원 받는 습관" / "토스에서 체크만 하면 돼요." (보상 → 쉬운 행동)
- "하루 9원 확정" / "토스에서 습관 체크해 보세요." (보상 확정 → 방법)

**감성·재미 앵글 — 캐릭터/감성 → 체험 유도:**
- "판다가 '맛있어~' 해요" / "토스에서 대나무 줘보세요." (귀여움 → 체험)
- "판다 밥 줄 시간이에요" / "토스에서 밥 주면 9원 받아요." (감성 → 보상)

**일상루틴 앵글 — 루틴 제안 → 보상 연결:**
- "아침 체크 한 번이면" / "토스에서 3원 받고 하루 시작해요." (루틴 → 보상)
- "저녁 습관 체크" / "토스에서 체크하면 9원 모아요." (루틴 → 누적 보상)

### 실패 패턴 분석 — 이런 소재는 클릭되지 않는다

**실패 유형 1: 타이틀 반복형** — 본문이 타이틀과 같은 말을 다시 한다.

| 타이틀 | 실패 본문 | 문제 진단 | 수정 본문 |
|--------|----------|----------|----------|
| "매일 9원 받기" | "토스에서 하루 9원 받아요." | "뭘 하면 9원을 받아?" 답 없음 | "토스에서 습관 체크하면 돼요." |
| "아침 저녁 체크" | "토스에서 아침·저녁 체크해요." | 타이틀을 그대로 문장화한 것뿐 | "토스에서 체크하면 매일 9원 받아요." |
| "판다 밥 주기" | "토스에서 판다 밥 주고 받아요." | "뭘 받아?" 목적어 불명확 | "토스에서 판다 밥 주면 9원 받아요." |

**실패 유형 2: 정보 공백형** — 행동도 보상도 구체적이지 않다.

| 타이틀 | 실패 본문 | 문제 진단 | 수정 본문 |
|--------|----------|----------|----------|
| "건강카드 도착" | "토스에서 뭐 눌러요." | "뭘 누르라는 거야?" 행동 불명확 | "토스에서 건강 체크하면 9원 받아요." |
| "하루 3번 체크" | "토스에서 버튼 3번 눌러요." | "눌러서 뭐?" 보상 없음 | "토스에서 3번 누르면 9원 받아요." |

**실패 유형 3: 보상 불명확형** — 보상이 있지만 구체적이지 않다.

| 타이틀 | 실패 본문 | 문제 진단 | 수정 본문 |
|--------|----------|----------|----------|
| "습관 3번 체크" | "토스에서 버튼 3번 눌러요." | 누르면 뭘 받는지 안 보임 | "토스에서 3번 누르면 9원 받아요." |
| "판다 밥 주기" | "토스에서 판다 밥 주고 받아요." | "받아요"가 뭘 받는지 불분명 | "토스에서 판다 밥 주면 9원 받아요." |

### 자가 체크 질문 (매 소재마다 적용)

타이틀을 쓴 뒤, 본문을 쓰기 전에 이 질문을 던진다:
1. "타이틀만 읽은 사람이 **'그래서?'**라고 물으면, 본문이 뭐라고 답하지?"
2. "본문에 **행동**(뭘 하면 되는지)이 들어있나, **보상**(뭘 받는지)이 들어있나, 둘 다 있나?"
3. "타이틀과 본문을 이어서 읽었을 때 **완성된 한 문장**처럼 들리나?"

답이 안 나오면 본문을 다시 쓴다.

### ONE MESSAGE 원칙 사례

| Loser (메시지 2개 혼합) | Winner (메시지 1개) |
|---|---|
| "뉴스 해석하는 법 / 주가를 올리는 뉴스를 골라봤어요" | "주가를 올리는 뉴스 / 5개로 요약했어요" |
| "기업 이익 보는 법 3가지 / 투자 기본기를 알려드려요" | "주식할 때 전문가가 꼭 보는 '이것' / 김토스님에게 알려드려요" |

### RADICAL SPECIFICITY 사례

**숫자의 힘:**
- "미션 도착" → "미션 **4개** 도착" → 전환율 **4배** 증가

**행동 구체화:**
- Loser: "숨은 쿠폰 5장 찾기 시작 / 김토스님은 몇 개까지 발견할 수 있을까요?"
- Winner: "숨은 쿠폰 5장 찾기 시작 / 토스쇼핑에서 **스크롤 해서** 찾아보세요" → CTR 3%↑

- Loser: "예민함 테스트를 해볼 수 있어요 / 확인하기 >" (앱인토스 실제 사례)
- Winner: "성격 테스트 도착 / **5개 질문에 답하기**" → CTR 3%↑

**대상 구체화:**
- Loser: "누를 때마다 돈 주는 서비스 / 확인하기 >" (앱인토스 실제 사례)
- Winner: "**돼지 저금통** 누르면 용돈 드려요 / 용돈 받으러 가기 >" → CTR 4%↑

### GUARANTEED REWARD 사례

| Loser | Winner | 성과 차이 |
|---|---|---|
| "**최대** 100만원 받기" | "**최소** 100원 뽑기" | 노출수 **20배**↑ |
| "**마음껏** 쿠폰 받아요 / 쇼핑 쿠폰 꾸러미를 확인해보세요" | "쿠폰 **1장은 무조건** 받아요 / 쇼핑 쿠폰 꾸러미를 확인해보세요" | 노출수 **5배**↑ |

작은 금액의 확실한 보상 효과:
- "오늘의 공짜 웹툰 보고 / **최소 2원** 받기" → CTR **30%** 이상
- "카드 OX 퀴즈 도전하고 / **최소 1원** 받기" → CTR **25%** 이상

## 짧게 쓰기 3가지 테크닉 상세

### 테크닉 1: 주제 생략

의도가 전달되면 주어나 부사구를 과감히 없앤다.
- AS IS: "전기차타면 **정부에서** 주는 돈" → TO BE: "전기차타면 **받는** 돈"

### 테크닉 2: 명사구 재구성

불필요한 조사·동사를 줄이되, 명사 3개 연속 나열은 피한다. 동사형 명사로 마무리.
- AS IS: "전투 **모자를 쓴** 고양이 **소환해봐요**" → TO BE: "전투 고양이 **소환하기**"

### 테크닉 3: 타이틀에 핵심 키워드만

- AS IS: "김토스님, 지금 차 번호 입력하면 / 50원 받아요. 입력하고 2월 말 시세도 확인 해보세요."
- TO BE: "**김토스님 차 번호** / 입력하면 50원 받아요."

- AS IS: "오늘은 청년희망적금 신청일이에요 / 준비물은 주민등록초본, 토스에서 받을 수 있어요."
- TO BE: "**청년희망적금** / 99년생은 오늘부터 신청할 수 있어요."

## 쉬운 말 변환 사전

| 어려운 말 | 쉬운 말 |
|----------|---------|
| 타임어택 | 시간 제한 |
| 최저가 | 제일 싼 |
| 보조금 | 정부에서 주는 돈 |
| 지역 | 동네, 우리 동네 |
| 혜택 비교 분석 | 뭐가 좋은지 |
| 신청 | 받기, 신청하기 |
| 프리미엄 | 좋은, 괜찮은 |
| 리워드 | 보상, 받는 것 |
| 이벤트 | (맥락에 따라) 행사, 기회 |
| 디펜스 | 방어 게임 |
| 타워디펜스 | 성 지키기 게임 |
| 퍼즐 | 맞추기 게임 |
| 캐주얼 | 간단한, 쉬운 |
| 시뮬레이션 | 체험 게임 |

**변환 검증법 — 소리 내서 읽기:**
1. 문구를 소리 내서 읽는다
2. 어색한 단어가 있으면 더 자주 쓰는 말로 교체
3. "내 지역 선택하면" → 소리 내서 읽으면 "동네"가 더 자연스럽다 → "내 **동네** 누르면"
4. "전기차 보조금" → "'보조금'이 어렵다" → "전기차타면 **정부에서 주는 돈**"

## Good 사례 모음 (이런 톤과 구조로 쓴다)

**주의:** 아래 사례는 A/B 테스트 원칙(구체적 행동, 확실한 보상, 쉬운 말 등)의 레퍼런스로, 토스 자체 서비스(토스쇼핑, 요기요 등)의 문구도 포함되어 있다. **앱인토스 미니앱용 소재를 생성할 때는 반드시 본문에 '토스'를 포함해야 한다.** 아래 사례 중 '토스'가 빠진 문구는 톤과 구조만 참고하고, 실제 생성 시에는 '토스에서', '토스로' 등을 자연스럽게 넣는다.

본문은 반드시 마침표(.)로 끝낸다. 타이틀은 마침표 없이 끝낸다.

- "김토스님, 심리 테스트로 나의 연애 유형을 알아보세요." — 행동 구체적, 경험 가치 명확
- "상자를 쌓는 게임이에요. 토스에서 도전해 보세요." — 행동 구체적, 서비스 명확
- "퀴즈를 풀고 광고 보면 최소 5원을 받을 수 있어요." — 행동+확실한 보상
- "내 사주를 볼 수 있는 서비스가 새로 나왔어요." — 경험 가치 구체적
- "쿠폰 5장 찾기 / 토스쇼핑에서 스크롤해서 찾아보세요." — 짧고 행동 구체적 (CTR 2.5배)
- "요기요 3천원 쿠폰 / 토스에서 요기패스 가입한 김토스님에게 드려요." — 핵심 키워드 타이틀 (CTR 2.2배)
- "쿠폰 1장은 무조건 받아요 / 쇼핑 쿠폰 꾸러미를 확인해보세요." — 확실한 보상 (노출 5배)
- "성격 테스트 도착 / 5개 질문에 답하기." — 행동 구체적 (앱인토스 실제 사례)
- "돼지 저금통 누르면 용돈 드려요 / 용돈 받으러 가기." — 대상 구체적 (CTR 4%↑)
- "시간 제한 고스톱 도전하고 / 포인트 5원 받기." — 쉬운 한글 (CTR 6배)
- "제일 싼 상품 찾고 2원 받아요 / 매일 모을 수 있어요." — 쉬운 한글 (노출 60배)
- "오늘의 공짜 웹툰 보고 / 최소 2원 받기." — 작은 확실한 보상 (CTR 30%)

### 토스 포함 적용 예시

위 Good 사례 중 '토스'가 빠진 문구를 앱인토스 미니앱용으로 변환하는 방법:

| 원본 (토스 미포함) | 앱인토스용 변환 (토스 포함) |
|---|---|
| "5개 질문에 답하기." | "토스에서 5개 질문에 답해보세요." |
| "용돈 받으러 가기." | "토스에서 용돈 받으러 가기." |
| "포인트 5원 받기." | "토스에서 포인트 5원 받기." |
| "매일 모을 수 있어요." | "토스에서 매일 모을 수 있어요." |
| "최소 2원 받기." | "토스에서 최소 2원 받기." |

핵심: "토스에서", "토스로", "토스 앱에서" 등을 본문에 자연스럽게 삽입한다. 타이틀에는 넣지 않아도 되지만, 본문에는 반드시 포함해야 반려되지 않는다.

## Bad 사례 모음 (반려되거나 성과 낮음)

- "포인트를 받을 수 있어요. 바로 확인해 보세요." → 서비스 불명확 (반려)

- "최대 100만원 받기" → "최소 100원 뽑기"에 노출 20배로 패배

## 앵글 다양성 — 같은 앱, 다른 5가지 소재

하나의 앱에서 5세트를 만들 때, 같은 패턴을 반복하면 A/B 테스트 의미가 없다. 각 세트가 서로 다른 앵글로 접근해야 한다.

**Bad — 동일 패턴 반복 (건강습관 앱):**
1. "습관 3번 체크 / 토스에서 9원 받아요." ← 보상
2. "습관 1번 체크 / 토스에서 3원 받아요." ← 보상 (같은 패턴)
3. "복권 긁기 / 토스에서 1원 받아요." ← 보상 (같은 패턴)
4. "판다 밥 30번 / 토스에서 밥 주고 1원 받아요." ← 보상 (같은 패턴)
5. "습관 34개 보기 / 토스에서 골라보세요." ← 약간 다르지만 밋밋함

문제: 5개 중 4개가 "토스에서 N원 받아요"로 끝남. 유저 머릿속에 장면이 안 그려지고, 어떤 앵글이 효과적인지 데이터 비교 불가.

**Good — 5가지 앵글 활용 (같은 건강습관 앱):**
1. **호기심**: "판다가 배고프대요 / 토스에서 판다에게 밥 줘보세요." ← 뭔 판다? 궁금
2. **행동묘사**: "버튼 3번 눌러 9원 / 토스에서 습관 체크해 보세요." ← 3번 누르면 9원? 쉽네
3. **확실한보상**: "매일 9원 받는 습관 / 토스에서 체크만 하면 돼요." ← 매일 9원 확실
4. **감성·재미**: "판다가 '맛있어~' 해요 / 토스에서 대나무 줘보세요." ← 귀엽다 보고싶다
5. **일상루틴**: "아침 체크 한 번이면 / 토스에서 3원 받고 하루 시작해요." ← 아침루틴으로 자연스럽게

핵심: 같은 앱이라도 호기심, 행동, 보상, 감성, 루틴 앵글로 각각 접근하면 유저 반응이 완전히 달라진다. 스마트 발송이 CTR 높은 소재에 자동 집중하므로, 다양한 앵글 = 최적 소재를 찾을 확률 상승.

## 종합 Before→After 변환 사례

**Before:** "지자체별 전기차 혜택 비교 분석 / 결과를 확인해보세요." (복잡, 어려운 한자어, 행동 불명확)

변환 과정:
1. 키워드 분해: 지자체별 / 전기차 / 혜택 / 비교 / 분석 / 결과
2. 핵심 2~3개 선택: 전기차, 혜택
3. 재조합: "전기차 혜택을 비교해요 / 토스에서 확인해보세요."
4. 쉬운 말: "전기차 타면 받는 지원금 / 토스에서 확인해보세요."
5. 소리 내서 읽기: "보조금" → "정부에서 주는 돈"

**After:** "전기차타면 정부에서 주는 돈 / 토스에서 확인해보세요." (간결, 쉬운 말, 서비스 명확)
