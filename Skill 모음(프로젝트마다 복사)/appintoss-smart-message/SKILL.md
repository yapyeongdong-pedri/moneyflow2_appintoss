# 앱인토스 스마트 발송 마케팅 소재 생성기

앱인토스 미니앱 파트너를 위한 광고성 푸시 메시지(타이틀+본문) 작성 스킬.
토스 공식 웨비나 방법론(UX 라이터 유아란 + 사업 개발 매니저 김성민)과 앱인토스 소재 가이드를 기반으로, **반려율 0%**와 **CTR 극대화**를 동시에 달성한다.

반드시 Phase 1 → 2 → 3 순서로 진행한다.

### 핵심 마인드셋

토스를 쓰는 모든 사용자에게 내 앱을 처음 소개한다는 마음으로 작성한다. 사용자는 이 푸시를 '토스'라는 금융앱의 맥락 안에서 받아들인다. 메시지가 토스답게 보이고 자연스럽게 이해되어야 한다.

### 출처

토스 공식 웨비나(UX 라이터 유아란 + 사업 개발 매니저 김성민) 방법론, [앱인토스 스마트 발송 콘솔 가이드](https://developers-apps-in-toss.toss.im/smart-message/console.md), 실제 A/B 테스트 데이터 기반.

## Phase 1: 컨텍스트 수집

스킬 시작 시 AskUserQuestion으로 사용자 상황을 파악한다. 초보자를 가정하고 친절하게 질문한다.

### 첫 번째 질문 — 3개를 동시에 묻는다

**질문 A: 앱 개발 상태**
- "앱 개발이 완료됐어요" — 이미 동작하는 미니앱이 있음
- "아직 아이디어만 있어요" — 앱 개발 전, 메시지부터 설계 (역퍼널 방식)

**질문 B: 토스포인트 프로모션 유무**
- "네, 포인트 지급이 있어요" → 후속으로 지급 금액을 물어본다
- "아니요, 포인트 없이 운영해요"
- "아직 잘 모르겠어요" → **프로모션 있는 버전 + 없는 버전 둘 다 생성**

**질문 C: 캠페인 목적**
- "서비스를 처음 소개해요" — 앱을 모르는 유저에게 알리기
- "새 콘텐츠/이벤트가 있어요" — 미션, 퀴즈, 시즌 이벤트 등
- "혜택/보상을 알리고 싶어요" — 포인트, 쿠폰 중심
- "업데이트를 알리고 싶어요" — 새 기능, 콘텐츠 추가
- "다시 와줬으면 좋겠어요" — 이탈 유저 재참여
- "잘 모르겠어요" → 소개+혜택 두 버전으로 생성

### 프로모션 금액 안내 (반드시 전달)

실제 A/B 데이터(두바이 쫀득 쿠키 사례):

| 문구 | CTR | 노출 |
|------|-----|------|
| **9원 받기** | **6.83%** | 314,693 |
| 11원 받기 | 6.02% | 267,621 |
| "포인트 받기" (금액 미명시) | 4.12% | 291 (노출 거의 안 됨) |

- **9원 또는 11원**을 강력 권장
- 단위는 "포인트" 대신 **"원"** 사용 (11포인트 5.78% < 11원 6.02%)
- 다른 금액을 원해도 이 데이터를 반드시 안내

### 두 번째 질문

**앱 완료 시:**
- 앱의 핵심 기능, 타겟 유저, 주요 보상/혜택, 한 줄 소개를 질문
- **눈에 보이는 요소** 수집: 캐릭터, 아이콘, 색상, 버튼 이름, 애니메이션, 말풍선, 재화명 등 — 이 디테일이 클릭을 만드는 소재의 원료다
- **유저의 구체적 동작**: 탭, 긁기, 스크롤, 체크, 누르기 등 실제 손가락 동작
- 가능하면 코드베이스를 탐색해서 앱 정보를 자동 수집 후 사용자에게 확인

**아이디어 단계 시:**
- 앱 종류 (게임/퀴즈/테스트/정보/금융 등)
- 사용자가 얻는 핵심 가치 또는 보상
- 타겟 사용자 (토스 3천만 유저 중 누구)
- 아이디어가 모호하면 → [역퍼널 브레인스토밍](references/reverse-funnel.md) 실행

### 소재 발상 — 규칙 적용 전에 반드시 먼저 수행

규칙을 적용하기 전에 **창의적 발상**이 먼저다. Phase 2에서 수집한 정보를 활용하되, 서비스를 처음 보는 사람의 눈으로 바라본다.

**발상 프로세스:**
1. **눈으로 보기**: 앱 화면에 보이는 캐릭터, 색상, 버튼, 애니메이션, 말풍선을 떠올린다
2. **손가락 동작 떠올리기**: 유저가 실제로 하는 동작(탭, 긁기, 체크, 스크롤)을 떠올린다
3. **5가지 앵글로 발산**: 아래 표의 각 앵글마다 최소 1개의 메시지 아이디어를 만든다
4. **클릭 욕구 필터링**: 각 아이디어를 읽고 "나라면 이거 눌러볼까?" 자문한다. 안 누를 것 같으면 탈락

**5가지 앵글 — 반드시 타이틀+본문 세트로 발상한다:**

| 앵글 | 타이틀 예시 | 본문 예시 | 연결 원리 |
|------|-----------|----------|----------|
| 호기심 | "판다가 배고프대요" | "토스에서 판다에게 밥 줘보세요." | 궁금증→행동 안내 |
| 행동묘사 | "버튼 3번 눌러 9원" | "토스에서 습관 체크해 보세요." | 행동+보상→구체적 안내 |
| 확실한 보상 | "매일 9원 받는 습관" | "토스에서 체크만 하면 돼요." | 보상→쉬운 행동 |
| 감성·재미 | "판다가 '맛있어~' 해요" | "토스에서 대나무 줘보세요." | 감성→체험 행동 |
| 일상루틴 | "아침 체크 한 번이면" | "토스에서 3원 받고 하루 시작해요." | 루틴→보상 연결 |

**왜 앵글 다양성이 중요한가:** 같은 "N원 받아요" 패턴을 5번 반복하면 A/B 테스트에서 비교할 대상이 없다. 서로 다른 앵글의 소재가 있어야 어떤 방향이 유저에게 먹히는지 데이터로 검증할 수 있다. 스마트 발송은 여러 소재 중 CTR이 높은 것에 자동으로 노출을 집중하므로, 다양한 앵글 = 더 높은 최종 CTR이다.

**그림 그리기 원칙:** "돈 주는 서비스" 대신 "판다를 톡톡 누르면 '맛있어~' 하고 대나무 먹어요"처럼, 유저 머릿속에 장면이 그려지는 문구가 CTR을 만든다. 추상적 설명 → 구체적 장면 변환이 핵심이다.

## 타이틀↔본문 연결 — 메시지 완성의 핵심

타이틀과 본문은 하나의 완성된 대화다. 타이틀이 미끼(hook)를 던지고, 본문이 낚시줄(answer)을 건다. 타이틀만 읽은 유저가 "그래서 뭐?", "어떻게?", "뭘 받는데?"라고 물을 때, 본문이 즉시 답해야 한다. **답이 없으면 클릭이 없다.**

**3가지 연결 공식:**
- **보상→행동**: 타이틀에 보상, 본문에 "어떻게 받는지" → "매일 9원 받는 습관" / "토스에서 체크만 하면 돼요."
- **행동→보상**: 타이틀에 행동, 본문에 "뭘 받는지" → "습관 3번 체크" / "토스에서 체크하면 9원 받아요."
- **호기심→행동**: 타이틀에 궁금증, 본문에 "뭘 하면 되는지" → "판다가 배고프대요" / "토스에서 판다에게 밥 줘보세요."

**캠페인 목적별 추가 공식** (상세 → [references/high-ctr-templates.md](references/high-ctr-templates.md)):
- **소개→접근성**: "건강 체크 서비스" / "토스에서 바로 써볼 수 있어요."
- **도착→사회적 증거**: "건강 미션 도착" / "토스에서 벌써 1만명이 도전했어요."
- **최소 노력→보상**: "체크만 해도 9원" / "토스에서 습관 체크해 보세요."
- **미수령→받기 방법**: "안 받은 9원" / "토스에서 체크하면 바로 받아요."

**절대 피할 실패 패턴 — 타이틀을 본문에서 반복하는 것:**

| 타이틀 | 실패 본문 | 문제 | 수정 본문 |
|--------|----------|------|----------|
| "매일 9원 받기" | "토스에서 하루 9원 받아요." | 보상만 반복, 행동 없음 | "토스에서 습관 체크하면 돼요." |
| "습관 3번 체크" | "토스에서 버튼 3번 눌러요." | 행동만 반복, 보상 없음 | "토스에서 3번 누르면 9원 받아요." |
| "판다 밥 주기" | "토스에서 판다 밥 주고 받아요." | "뭘 받아?" 불명확 | "토스에서 판다 밥 주면 9원 받아요." |
| "건강카드 도착" | "토스에서 뭐 눌러요." | 행동·보상 모두 불명확 | "토스에서 건강 체크하면 9원 받아요." |
| "아침 저녁 체크" | "토스에서 아침·저녁 체크해요." | 타이틀 그대로 반복 | "토스에서 체크하면 매일 9원 받아요." |

자가 체크: **"타이틀만 읽은 사람이 '그래서?'라고 물으면, 본문이 즉시 답하는가?"**

## 글자수 세기 — 가장 중요한 규칙

**띄어쓰기(공백)를 포함해서 센다.** 한글, 숫자, 영문, 특수문자, 띄어쓰기 모두 각각 1자다. 마침표(.)도 1자다. 본문 글자수를 셀 때 마침표까지 포함한다.

**한 글자씩 번호를 매겨서 세는 것이 유일하게 정확한 방법이다:**

```
"고양이 전쟁 게임" = 고(1)양(2)이(3) (4)전(5)쟁(6) (7)게(8)임(9) = 9자
"성격 테스트 도착" = 성(1)격(2) (3)테(4)스(5)트(6) (7)도(8)착(9) = 9자
"내 성격 유형 결과" = 내(1) (2)성(3)격(4) (5)유(6)형(7) (8)결(9)과(10) = 10자 ← 7~9자 초과!
"토스에서 9원 받아요." = 토(1)스(2)에(3)서(4) (5)9(6)원(7) (8)받(9)아(10)요(11).(12) = 12자
```

"내 성격 유형 결과"를 띄어쓰기 빼고 세면 7자처럼 보이지만, 실제로는 10자다. **이 실수가 가장 흔하다.** 타이틀/본문을 쓸 때마다 반드시 한 글자씩 번호를 매겨서 검증한다.

**글자수 제한 요약:**

| 항목 | Tier 1 | Tier 2 |
|------|--------|--------|
| 타이틀 | 13자 이내 (공백·마침표 포함) | 7~9자 (공백 포함, 10자 이상 절대 불가) |
| 본문 | 20자 이내, 해요체 문장 (공백·마침표 포함) | 20자 이내, 해요체 문장 |

**"(광고)" 접두사 주의:** 광고성 푸시는 플랫폼이 본문 앞에 `(광고)` 접두사를 자동으로 추가한다. 사용자가 작성하는 본문 20자에는 포함되지 않지만, 실제 화면에서는 "(광고) " 4글자+공백이 본문 앞에 붙어 표시된다. 큰글자모드에서 본문이 잘릴 수 있으므로 20자 이내를 지키되, 해요체(~요) 문장으로 자연스럽게 쓴다.

### 본문은 해요체 문장으로 쓴다 — 글자수만큼 중요한 규칙

본문은 자연스러운 한국어 문장이어야 한다. 토스 푸시의 기본 톤은 **해요체**(~요, ~세요, ~어요)다. 명사구 파편("토스에서 3번.", "토스에서 복권 1장.")은 기계가 쓴 것처럼 느껴져 클릭을 유도하지 못한다. 글자수를 줄이느라 문장을 깨뜨리면 안 된다 — 20자 안에서 해요체 문장은 충분히 가능하다.

| 나쁜 예 (명사구 파편) | 좋은 예 (해요체 문장) | 글자수 |
|---|---|---|
| "토스에서 습관 3번." (11자) | "토스에서 습관 체크해 보세요." (15자) | 15자 |
| "토스에서 복권 1장." (11자) | "토스에서 복권 1장 긁어보세요." (16자) | 16자 |
| "토스에서 판다 3번." (11자) | "토스에서 판다에게 밥 줘보세요." (16자) | 16자 |
| "토스에서 3번 누르기." (12자) | "토스에서 3번 눌러보세요." (14자) | 14자 |
| "토스에서 34개 골라요." (13자) | "토스에서 34개 골라보세요." (14자) | 14자 |

**해요체 마무리 패턴:** ~해 보세요, ~받아요, ~해요, ~있어요, ~줘보세요, ~골라보세요, ~도전해요

## 반려 방지 — 절대 규칙

앱인토스 소재 반려율이 40~50%다. 아래 6가지를 반드시 피한다.

| 순위 | 반려 사유 | 금지 사항 |
|------|----------|----------|
| 1 | 광고성 표현 | 이모지, 느낌표(!), "지금 바로", "놓치지 마세요", "특별한", "대박" |
| 2 | 규격 위반 | 타이틀 13자 초과, 본문 20자 초과, 맞춤법 오류 (**띄어쓰기 포함해서 센다**) |
| 3 | 서비스 예상 불가 | "혜택을 확인해 보세요" — 앱이 뭔지 모름 |
| 4 | 유행어/은어 | 밈, 명대사, 유행어, 특정 그룹만 아는 은어 |
| 5 | 유저 반감 | 불안감("늦으면 사라져요"), 자극적/민감 소재 |
| 6 | '토스' 미포함 | 본문에 '토스'가 반드시 포함되어야 함 |

**캠페인 유형 주의:** 이 스킬은 **광고성 메시지** 전용이다. 기능성 메시지(거래 알림, 배송 완료 등 서비스 필수 정보)에는 사용하지 않는다. 광고성 메시지는 토스 앱의 마케팅 수신 동의 사용자에게만 발송되며, 소재와 성과 중심으로 자동 최적화된다. 기능성 메시지가 필요하면 앱인토스 콘솔의 기능성 캠페인 메뉴와 API를 사용한다.

**추가 금지 패턴:**
- `{{ userName }}님` 이름 변수
- "전문가급", "최저가" 같은 근거 없는 과장
- "포인트를 받을 수 있어요. 바로 확인해 보세요." — 서비스 불명확
- "월급은 올랐나요?" — 민감 소재

**문장 부호 규칙 (반드시 준수):**
- **본문은 반드시 마침표(.)로 끝낸다.** 예: "토스에서 3판 하고 9원 받아요." (O) / "토스에서 3판 하고 9원 받아요" (X)
- **타이틀은 마침표 없이 끝낸다.** 예: "고양이 게임 도착" (O) / "고양이 게임 도착." (X)
- 본문은 해요체(~요/~세요)로 자연스럽게 끝낸 뒤 마침표를 붙인다: "토스에서 도전해 보세요." "토스에서 9원 받아요."

**검토 프로세스:** 작성된 소재는 앱인토스 콘솔에서 검토 요청 후 영업일 기준 2~3일 소요된다. 검토를 받지 않은 메시지는 발송 불가하므로, 이벤트 일정에 맞춰 미리 소재를 준비하도록 사용자에게 안내한다.

## 라이팅 4원칙 + 짧게 쓰기

### 원칙 1: ONE MESSAGE — 하나의 핵심만 전달

하나의 푸시에 2개 이상 메시지를 섞지 않는다.

자가 체크: "이 푸시에 메시지가 2개 이상 섞여 있지 않나?"

### 원칙 2: RADICAL SPECIFICITY — 유저 머릿속에 그림 그려주기

구체적으로 쓰는 이유: 추상적인 말("혜택 확인")은 머릿속에 아무 장면도 만들지 못한다. 반면 "돼지 저금통 누르면 용돈 드려요"는 돼지 모양 버튼을 탭하는 장면이 바로 떠오른다. **유저 머릿속에 장면이 그려져야 손가락이 움직인다.**

- **숫자**: "미션 도착" → "미션 4개 도착" — 전환율 4배
- **행동**: "확인하기" → "5개 질문에 답하기" — CTR 3%↑
- **대상 (가장 중요)**: "돈 주는 서비스" → "돼지 저금통 누르면 용돈 드려요" — CTR 4%↑

대상 구체화가 CTR에 가장 큰 영향을 준다. Phase 2에서 수집한 눈에 보이는 요소(캐릭터, 아이콘, 색상, 말풍선 등)를 적극 활용한다.

구체적 행동 패턴: 버튼 누르고, 질문에 답하고, 스크롤해서, 아래로 내려서, 톡톡 눌러서, 긁어서
구체적 숫자 패턴: 미션 2개 하고, 게임 5판 하고, 3개 보기, 10개 모으기

자가 체크: "이 문구를 읽으면 유저 머릿속에 구체적 장면이 그려지는가?"

### 원칙 3: GUARANTEED REWARD — 확실한 보상

"최소 보상"이 "최대 혜택"을 압도적으로 이긴다.

핵심 데이터:
- "최대 100만원 받기" vs "최소 100원 뽑기" → 최소가 **노출 20배**
- "마음껏 쿠폰 받아요" vs "쿠폰 1장은 무조건 받아요" → 무조건이 **노출 5배**
- 1~2원이라도 "최소"라는 확실한 보상이면 CTR 20~30%

**보상 문구 작성 규칙:**
1. 구체적 금액 + "원" 단위: "9원 받기", "11원 받기"
2. "혜택"/"보상"/"포인트" 같은 막연한 단어 금지
3. "최대 N원" 대신 "최소 N원" 또는 "N원 받기"
4. "무조건", "1장은", "최소" 확실성 표현 활용
5. 프로모션 없으면 경험 가치: "내 연애 유형 알아보기", "오늘의 운세 보기"

자가 체크: "유저가 '나도 확실히 받을 수 있겠다'고 느끼는가?"

### 원칙 4: COMMON WORDS — 쉬운 말로 풀어쓰기

외래어/한자어를 일상 한글로 바꾸면 CTR이 급상승한다.

| 어려운 말 | 쉬운 말 | 효과 |
|----------|---------|------|
| 타임어택 | 시간 제한 | CTR 6배↑ |
| 최저가 | 제일 싼 | 노출 60배↑ |
| 보조금 | 정부에서 주는 돈 | |
| 지역 | 동네, 우리 동네 | |
| 혜택 비교 분석 | 뭐가 좋은지 | |
| 신청 | 받기, 신청하기 | |
| 디펜스 | 방어 게임 | |
| 타워디펜스 | 성 지키기 게임 | |
| 퍼즐 | 맞추기 게임 | |
| 시뮬레이션 | 체험 게임 | |

검증법: 소리 내서 읽었을 때 입에서 자연스러우면 OK. 어색하면 더 자주 쓰는 말로 교체.

자가 체크: "이 문구를 소리 내서 읽었을 때 자연스러운가?"

### +α 짧게 쓰기 — 타이틀에 집중 적용

타이틀은 짧을수록 CTR이 높다 (데이터 검증, 2배 이상 상승). 본문은 20자 이내에서 해요체 문장을 유지한다 — 글자수를 줄이느라 명사구 파편("토스에서 3번.", "토스에서 1장.")이 되면 마케팅 효과가 없다.

**Tier 2에서도 본문 규칙은 Tier 1과 동일하다.** 짧게 쓰기는 타이틀에만 적용한다. 본문은 반드시 **해요체(~요/~세요) + 마침표(.)**를 유지한다. 또한 5세트의 본문은 모두 서로 달라야 한다 — 같은 본문 반복은 A/B 테스트 의미를 없앤다.

| Tier 2 본문 ✗ (명사구 파편) | Tier 2 본문 ✓ (해요체+마침표) |
|---|---|
| "토스에서 9원 받기" | "토스에서 9원 받아요." |
| "토스에서 5개 답하고 9원 받기" | "토스에서 5개 답하면 9원 받아요." |
| "토스에서 심리 테스트 9원 받기" | "토스에서 테스트하면 9원 받아요." |

**타이틀 최적 길이: 7~9자**
- 토스 유저 67%가 안드로이드, 그 중 25%가 큰글자모드
- 7자 넘으면 큰글자모드에서 잘려서 "..."으로 표시됨

**3가지 테크닉:**
1. **주제 생략**: 의도가 전달되면 과감히 없앤다 ("정부에서 주는 돈" → "받는 돈")
2. **명사구 재구성**: "모자를 쓴 고양이 소환해봐요" → "전투 고양이 소환하기" (명사 3연속 나열은 피함)
3. **타이틀에 키워드만**: "김토스님, 지금 차 번호 입력하면" → "김토스님 차 번호"

## Tier 2 전용 CTR 극대화 팁

1. **타이틀은 반드시 7~9자** (띄어쓰기 포함, 큰글자모드 잘림 방지). 10자 이상은 Tier 2 자격 미달이므로 절대 불가. 9자에 맞추기 어려우면 핵심 키워드 1개만 남기고 나머지를 본문으로 이동한다.
2. 불필요한 주어/조사/동사 제거
3. 명사구 재구성
4. 타이틀 = 핵심 키워드 1~2개
5. "최소 N원" 또는 "N원 받기"
6. "무조건", "1장은" 확실성 표현
7. 타이틀에서 뺀 정보는 본문으로

**빈출 키워드별 7~9자 타이틀 예시 (띄어쓰기 포함 글자수):**

심리테스트/성격 앱:
- "성격 테스트 도착" = 9자 ✓ / "내 성격 유형은" = 8자 ✓ / "연애 유형 보기" = 8자 ✓
- "내 성격 유형 결과" = 10자 ✗ (띄어쓰기 3개 때문에 초과!)
- "성격 테스트하고" = 8자 ✓ / "연애 유형 테스트" = 9자 ✓

게임 앱:
- "고양이 전쟁 게임" = 9자 ✓ / "전투 고양이 소환" = 9자 ✓ / "9원 받는 게임" = 8자 ✓

운세/사주 앱:
- "내 운세 도착" = 7자 ✓ / "오늘의 연애운" = 7자 ✓ / "내 재물운 보기" = 8자 ✓

**축약이 필요한 위험 패턴 (띄어쓰기 포함하면 10자 이상 되는 것들):**
- "내 성격 유형 결과" (10자) → "내 성격 유형은" (8자) 또는 "성격 유형 보기" (8자)
- "내 연애 유형 결과" (10자) → "연애 유형 보기" (8자) 또는 "내 연애 유형은" (8자)
- "성격 유형 알아보기" (10자) → "성격 알아보기" (8자) 또는 "내 성격 유형은" (8자)
- "성격 테스트하고 9원" (12자) → "성격 테스트 9원" (9자)

**실제 실패 사례 — 이 패턴을 반복하지 않는다:**

| 실패한 타이틀 | 모델이 센 글자수 | 실제 글자수 (공백 포함) | 원인 |
|---|---|---|---|
| "5개 답하면 아는 나" | 9자 | **11자** | 공백 3개를 빼고 셈 |
| "성격 유형 나오는 곳" | 9자 | **11자** | 공백 3개를 빼고 셈 |
| "질문 5개 내 성격" | 8자 | **10자** | 공백 3개를 빼고 셈 |

**공통 원인:** 한글·숫자만 세고 공백을 빼면 7~9자로 보이지만, 공백을 포함하면 10자 이상이 된다. **공백이 3개 이상인 타이틀은 Tier 2 부적격일 가능성이 높다.** Tier 2 타이틀은 공백 2개 이하를 권장한다. 반드시 번호를 매겨서 확인한다.

## 자가검증 체크리스트

모든 소재를 생성한 후, **각 소재마다 하나씩** 아래 검증을 수행한다. 특히 글자수 검증(항목 3~4)은 반드시 한 글자씩 번호를 매겨서 확인한다.

**[필수] 글자수 검증 — 각 타이틀과 본문에 대해 번호를 매긴다:**
```
타이틀: "고양이 전쟁 게임" = 고(1)양(2)이(3) (4)전(5)쟁(6) (7)게(8)임(9) = 9자 ✓
본문: "토스에서 3판 도전해요." = 토(1)스(2)에(3)서(4) (5)3(6)판(7) (8)도(9)전(10)해(11)요(12).(13) = 13자 ✓
```
위반이 발견되면 해당 소재를 수정한 뒤 다시 번호를 매겨 재검증한다.

**체크리스트:**
1. 반려 사유 6가지에 하나도 해당하지 않는가
2. '토스'가 본문에 자연스럽게 포함되어 있는가
3. **[글자수 번호 매기기 필수]** Tier 1: 타이틀 13자 이내, 본문 20자 이내 (공백·마침표 포함)
4. **[글자수 번호 매기기 필수]** Tier 2: 타이틀 7~9자 (공백 포함, 10자 이상이면 축약 후 재검증)
5. 본문이 마침표(.)로 끝나는가 (타이틀은 마침표 없이)
6. 외래어/한자어 대신 쉬운 한글을 사용했는가
7. 처음 보는 사람도 무슨 앱인지 바로 이해할 수 있는가
8. 프로모션 금액은 "원" 단위로 표기되었는가
9. "포인트 받기" 같은 막연한 보상 표현이 없는가
10. "최대 N원" 대신 "최소 N원" 또는 확정 금액인가
11. 하나의 메시지만 담았는가
12. 유저 행동이 구체적으로 그려지는가
13. 본문이 해요체(~요/~세요)로 끝나는 자연스러운 문장인가 (명사구 파편 금지)
14. 이모지, 느낌표가 없는가
15. **[클릭 욕구]** 이 푸시를 받았을 때 "뭔데? 눌러볼까?" 하는 호기심이 드는가 (단순히 읽기 쉬운 것 ≠ 누르고 싶은 것)
16. **[앵글 다양성]** 5세트가 각각 다른 앵글(호기심/행동묘사/확실한보상/감성·재미/일상루틴)을 사용하는가 — "N원 받아요" 같은 동일 패턴 반복이면 실패
17. **[타이틀↔본문 연결]** 타이틀이 던진 "그래서 뭐?"에 본문이 답하는가 — 타이틀을 본문에서 반복하거나, 행동만 있고 보상이 없거나, 보상만 있고 행동이 없으면 실패
18. **[본문 다양성]** 5세트의 본문이 모두 서로 다른가 — 동일 본문 반복은 A/B 테스트 의미를 없앤다

**[Tier 2 재검증 루프 — 생략 불가]** 모든 소재 생성이 끝나면, Tier 2 타이틀만 다시 한 글자씩 번호를 매겨서 재검증한다. 10자 이상이 하나라도 있으면 즉시 수정 후 재검증한다. 또한 본문이 해요체+마침표가 아닌 것이 있으면 수정한다. 이 단계를 건너뛰면 반드시 글자수 오류가 발생한다.
